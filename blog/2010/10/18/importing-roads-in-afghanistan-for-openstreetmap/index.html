Importing Roads in Afghanistan for OpenStreetMap

Oct 18th, 2010

I never thought that I’d curse myself for encouraging people to add data to OpenStreetMap. Recently though I was regretting it a little because I had the road data from Afghanistan Information Management Services. Through negotiation between AIMS and Todd Huffman they changed their data license to ODbL to allow import into OpenStreetMap. We were also giving specific permission to import the data into OSM. There is a shortage of usable road data in the country for those who are non-military to use. Being able to navigate quickly to the correct place can be a safety issue and allows for more effect reconstruction and development efforts.

Since there was already a decent amount of mapping in Kabul and Jalalabad it was important during the import not to remove that data. At least a portion of it was collected by GPS and was more up to date than the data being imported. Thea Clay and I worked together on this process to breakup the data, convert it to OSM format and then manually verify it. In order to accomplish this we first used Quantum GIS to slice all the roads by province. I think took those files and utilized shp-to-osm in order to create OSM files. After this one by one we pulled each of the files into JOSM to compare the data to that already available and determine which data to keep and which to import. At times I also utilized Yahoo imagery and LandSat imagery to determine what was the appropriate action as well. The issue of importing data when already existing information is there is a difficult problem. Fortunately we were able to do this manually for a much larger country such as the United States this definitely wouldn’t be a two person job.

The exact steps we used were:

  1. Downloaded Roads and Provinces for Afghanistan as Shapefiles
  2. Opened up datasets in Quantum GIS
  3. Vector -> Data Management Tools -> Join Attributes by Location
  4. Vector -> Data Management Tools -> Split Vector Layer
  5. Create rules file for Shp-to-OSM
  6. Run each shapefile through Shp-to-OSM with rules file.
  7. Individually open each OSM file in JOSM, download the data in OSM to verify data does not exist.
  8. Usually see this error

  1. Verify Data
  2. Merge import data into downloaded data
  3. Upload to OSM (sometimes here I had to resolve conflicts)

Things to note is we created a separate OSM user for this import. There was not data in many areas, but the areas that had been heavily edited required a lot of manual verification.

I hope that having a freely available road dataset for Afghanistan will help people living and working there. Maybe it will even encourage them to start adding their data to OpenStreetMap.
